# 게임플레이 오브젝트 컨셉

## 목적

수집물, 위험물, 골, 발판이 게임 규칙을 설명 없이 전달하도록 형태와 움직임을 고정한다.

실제 제작 에셋 목록과 모델링용 3면도 reference는 `gameplay_object_asset_pipeline.md`를 따른다.

## 플레이 레이어 소유권

환경 문서에서 말하는 `플레이 레이어`의 실제 상호작용 오브젝트는 이 문서가 소유한다. `environment_concept.md`는 배경, 근경/중경/원경, 조명 대비를 다루고, 아래 오브젝트의 형태/색/충돌 판독 기준은 여기에서 관리한다.

| 플레이 레이어 요소 | 소유 문서 | 환경 문서의 역할 |
|---|---|---|
| 발판 / 이동 발판 / 높이 조절 장치 | 이 문서와 `gameplay_object_asset_pipeline.md` | 배경이 발판 상단면, 이동 방향, 높낮이 조절 장치를 가리지 않게 함 |
| 별 조각 / 에너지 공 | 이 문서와 `gameplay_object_asset_pipeline.md` | 배경 별빛과 장식광이 수집물보다 튀지 않게 함 |
| 위험 장치 / 위험 범위 | 이 문서와 `gameplay_object_asset_pipeline.md` | 배경 기어/장치가 위험물과 혼동되지 않게 함 |
| 골 게이트 | 이 문서와 `gameplay_object_asset_pipeline.md` | 스테이지 끝 목표가 배경에 묻히지 않게 함 |
| 루미 | `../characters/lumi_character_concept.md` | 배경 명도/채도를 낮춰 캐릭터 실루엣을 보존함 |

플레이 레이어는 화면에서 가장 선명해야 한다. 발판, 수집물, 위험물, 골, 루미가 근경 장식이나 배경보다 먼저 읽혀야 하며, 색을 제거해도 역할이 구분되어야 한다.

## 별 조각

| 항목 | 기준 |
|---|---|
| 역할 | 기본 수집물, 메인 경로 안내 |
| 형태 | 작은 5각 별 전체가 아니라 깨진 별 파편 느낌 |
| 색상 | 금색, 밝은 노랑 |
| 재질 | 투명한 보석과 빛나는 금속 사이 |
| 움직임 | 제자리에서 천천히 회전, 작은 반짝임 |

| 금지 | 동전처럼 납작한 원형, 너무 큰 별 캐릭터 얼굴 |

### 발판 키트 구성

발판 키트는 하나의 막연한 `platform`이 아니라, 아래 파츠를 조립해서 정적 발판, 이동 발판, 높이 조절 발판에 재사용하는 기본 gameplay kit다.

| 파츠 | 에셋명 | 정의 | 사용 방식 |
|---|---|---|---|
| 1칸 기본 발판 | `SM_SR_Platform_1x1_A` | 플레이어가 한 번 착지하고 점프 타이밍을 잡을 수 있는 기본 착지면. 석재 top panel, 남색 금속 frame, 금색 corner cap이 함께 있는 slab | 짧은 착지 지점, 리프트 상단, 작은 이동 발판 |
| 2칸 기본 발판 | `SM_SR_Platform_1x2_A` | 1칸 발판을 두 칸 길이로 확장한 안정적인 착지면 | 초반 튜토리얼 발판, 골 앞 안전 구간, 긴 이동 발판 |
| 가장자리 trim | `SM_SR_Platform_Edge_A` | 발판 side를 읽기 쉽게 만드는 좁은 마감 파츠 | 상단 충돌면의 끝을 시각적으로 표시 |
| 모서리 cap | `SM_SR_Platform_Corner_A` | 금색 모서리 보호구/장식 파츠 | 발판의 방향과 크기를 작게 봐도 읽히게 함 |

`SM_SR_Platform_1x1_A`와 `SM_SR_Platform_1x2_A`는 실제 충돌면을 가진 발판 본체다. `SM_SR_Platform_Edge_A`와 `SM_SR_Platform_Corner_A`는 본체를 보강하는 시각 파츠이며, 단독으로 배치되는 gameplay 발판이 아니다.

## 발판 높이 조절 시스템

| 항목 | 기준 |
|---|---|
| 역할 | 발판의 높낮이를 바꾸는 게임플레이 장치. 수직 이동, 단계식 높이 변경, 타이밍 도전을 표현 |
| 제작 방식 | `고정 베이스`, `가변 리프트 컬럼`, `상단 마운트`, `발판 상단 모듈`을 조립하는 시스템으로 만든다 |
| 형태 | 고정 석재/금속 베이스, 망원경식 2~3단 컬럼, 양옆 가이드 레일, 기어 박스, 높이 게이지, 상단 발판 소켓 |
| 색상 | 발판과 같은 남회색/금색 베이스, 움직임 표시는 청록 발광 |
| 재질 | 공방 금속, 석재 베이스, 유리/에너지 게이지 |
| 움직임 | 베이스는 고정되고, 상단 마운트와 발판만 `MinHeight`와 `MaxHeight` 사이를 이동한다 |
| 충돌 | 실제 발판 충돌은 상단 발판이 담당한다. 높이 조절 장치 본체는 플레이어 이동을 불필요하게 막지 않게 단순 충돌 또는 장식 충돌로 처리 |
| VFX | 이동 중 청록 에너지 흐름, 도착 시 짧은 정지 pulse |
| 금지 | 배경 장식용 기어처럼만 보이는 형태, 높이 변화 방향이 읽히지 않는 형태, 발판 상단면을 가리는 과한 장식 |

### 높이 조절 구성

`gameplay_platform_lift_actuator_orthographic_v001.png`는 단일 리프트 장치의 외형만 보여줘 현재 구조를 충분히 설명하지 못한다. `gameplay_platform_lift_system_reference_v002.png`는 이동 범위를 보여주는 반투명 guide가 실제 에셋처럼 보일 수 있어 모델링 기준에서 제외했다.

현재 기준은 `gameplay_object_asset_pipeline.md`의 Static Mesh별 concept/orthographic reference다. `gameplay_platform_lift_system_tripo3d_orthographic_v003.png`는 `images/legacy/modeling_reference_pre_mesh_split/`로 이동된 과거 묶음형 reference이며, 현재 모델링 기준으로 사용하지 않는다. 실제 게임에서는 아래 구성이 함께 필요하다.

| 구성 | 역할 | 비고 |
|---|---|---|
| 발판 상단 모듈 | 플레이어가 실제로 밟는 충돌면 | `SM_SR_Platform_1x1_A` 또는 `SM_SR_Platform_1x2_A` 사용 |
| 상단 마운트 | 발판과 리프트를 연결하는 이동 소켓 | `TopMount` SceneComponent, 발판과 함께 이동 |
| 내부 리프트 컬럼 | 실제 높이 변화가 보이는 가변 기둥 | 위로 올라갈수록 노출되는 2~3단 telescope segment |
| 외부 가이드 레일 | 발판이 수직으로만 움직인다는 시각 정보 | 좌우 2개 레일, 고정 베이스와 상단 마운트를 연결 |
| 기어 박스 | 리프트 구동 장치 | 이동 중 회전하는 큰 기어와 작은 보조 기어 |
| 에너지 게이지 | 현재 높이/동작 상태를 보여주는 청록 발광 요소 | 베이스와 컬럼 옆에 세로 게이지 |
| 고정 베이스 | 바닥/벽에 고정되는 기준점 | 움직이지 않는 석재/금속 덩어리 |
| 최소/최대 높이 기준 | Blueprint 이동 범위 | editor gizmo, marker, numeric property로 표현 |
| Blueprint 이동 로직 | 발판 그룹을 위아래로 이동 | 메시가 아니라 `BP_SR_PlatformLiftActuator`의 책임 |

즉, `발판 높이 조절 장치`는 하나의 메시가 아니라 `발판 + 리프트 파츠 + Blueprint 이동 로직`으로 완성된다.

### 시각 구조 기준

| 상태 | 화면에서 보여야 할 것 |
|---|---|
| 낮은 위치 | 컬럼이 짧고 상단 발판이 베이스에 가까움 |
| 높은 위치 | 내부 컬럼 segment가 길게 노출되고 발판이 위에 있음 |
| 이동 중 | 기어가 회전하고 청록 에너지 게이지가 켜짐 |
| 정지 상태 | 게이지가 약하게 켜지고 발판 상단면이 명확함 |

리프트는 “발판을 받치는 배경 기둥”이 아니라 “발판이 움직일 것임을 미리 알려주는 장치”여야 한다. 따라서 컬럼, 레일, 게이지는 발판 아래에서 가려지지 않아야 한다.

### Tripo3D 모델링 기준

| 기준 | 설명 |
|---|---|
| 실제 메시만 표시 | ghost guide, 반투명 이동 경로, 화살표, 텍스트는 모델링 reference에 넣지 않는다 |
| 단일 포즈 | 하나의 실제 조립체를 정면/측면/상면으로 보여준다 |
| 기본 높이 | 컬럼과 가이드가 보이는 중간 높이 상태를 기본 모델링 포즈로 사용한다 |
| 파츠 분리 | PlatformMesh, TopMount, LiftBase, LiftColumn, GuideRail, GearBox, EnergyGauge가 읽혀야 한다 |
| 충돌면 | 발판 상단면은 장식 없이 명확해야 한다 |

## 체크리스트

- 색을 빼도 수집물/위험물/골이 구분되는가?

- 별 조각은 경로 안내처럼 배치 가능한 크기인가?

- 골 게이트는 멀리서도 도착지로 보이는가?
