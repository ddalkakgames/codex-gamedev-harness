# 기술 설계

상태: 초안
목적: 게임디자인 문서와 시스템 설계를 언리얼 구현 구조로 변환한다.

## 출처

- 게임디자인 문서: `../design/game_design_document.md`
- 시스템 설계: `../systems/`
- 제작 추적표: `../production/traceability_matrix.md`

## 아키텍처 원칙

- 먼저 플레이 가능한 증명을 만든다.
- Blueprint와 C++의 경계를 명확히 둔다.
- 데이터로 조정할 값과 코드로 고정할 값을 구분한다.
- GameDevMCP는 구현 자동화와 검증 자동화에 사용한다.

## 런타임 시스템

| 시스템 | 언리얼 구현 형태 | 상태 |
|---|---|---|
| 핵심 반복 구조 | 미정 | 초안 |
| 캐릭터 조작 | 미정 | 계획 |
| 도전 / 전투 | 미정 | 계획 |
| 성장 / 보상 | 미정 | 계획 |
| UX / UI | 미정 | 계획 |

## Blueprint / C++ 경계

| 영역 | Blueprint | C++ | 이유 |
|---|---|---|---|
| 프로토타입 게임플레이 | 예 | 선택 | 반복 속도 |
| 핵심 런타임 프레임워크 | 미정 | 미정 | 미정 |
| 데이터 작성 | 예 | 선택 | 디자이너 반복 작업 |
| 성능에 민감한 로직 | 아니오 | 예 | 미정 |

## GameDevMCP 자동화 후보

- 에셋 생성/수정/삭제
- Blueprint class/component/graph 작성
- UI 구성
- 머티리얼/나이아가라 임시 제작
- 레벨 actor 배치
- map check 검증
- compile/save 파이프라인

## 검증 hook 후보

- 컴파일 검증
- map check
- actor/component 존재 확인
- data table 검증
- PIE 경로 테스트
- 스크린샷 또는 viewport 근거

## 열려 있는 기술 결정

- 기준 프로젝트 템플릿
- 플레이어 컨트롤러 기준
- 카메라 시스템
- 저장/세션 모델
- 데이터 작성 방식
- 자동화 테스트 전략
