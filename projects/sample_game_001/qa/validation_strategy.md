# 검증 전략

상태: 초안
목적: AI가 만든 결과를 "만들었다"가 아니라 "게임디자인 의도대로 동작한다"로 검증한다.

## 검증 단계

| 단계 | 대상 | 근거 |
|---|---|---|
| 문서 검증 | 컨셉/GDD/시스템의 일관성 | 검토 체크리스트 |
| 작업 항목 검증 | 단일 구현 작업 | 컴파일, 점검, 보고서 |
| 시스템 검증 | 구성요소 통합 | 시스템 시나리오 테스트 |
| 세로 단면 검증 | 플레이 가능한 반복 구조 | PIE, 자동화, 플레이테스트 근거 |
| 버전 검증 | 버전이 약속한 경험 | 검증 기준과 알려진 문제 |
| 릴리즈 검증 | 배포 품질 | 패키징, QA, 릴리즈 노트 |

## 첫 플레이 가능 버전 검증

`v0.2_first_playable`은 아래를 증명해야 한다.

- 프로젝트가 열린다.
- 테스트 레벨이 로드된다.
- 플레이어가 시작할 수 있다.
- 플레이어가 핵심 행동을 수행할 수 있다.
- 도전 또는 장애 요소가 있다.
- 피드백 또는 보상이 보인다.
- 목표 조건에 도달할 수 있다.
- 알려진 문제가 문서화되어 있다.
