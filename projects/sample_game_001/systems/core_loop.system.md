# 시스템 설계: 핵심 반복 구조

상태: 초안
출처: `../design/game_design_document.md`

## 설계 의도

핵심 반복 구조는 플레이어가 게임에서 반복적으로 수행하는 가장 작은 의미 있는 행동 순환이다.

아직 게임 컨셉이 선택되지 않았으므로 구체 규칙은 미정 상태다.

## 관련 게임 설계 원칙

| 게임 설계 원칙 | 관계 |
|---|---|
| `principle_01` | 미정 |
| `principle_02` | 미정 |

## 반복 구조 정의

```text
미정
```

예시 형식:

```text
상황 파악 -> 판단 -> 행동 -> 위험 해결 -> 피드백/보상 획득 -> 다음 상황으로 이동
```

## 시스템 구성요소

| 구성요소 | 목적 | 상태 | 파생 작업 |
|---|---|---|---|
| 플레이어 핵심 행동 | 플레이어가 반복하는 가장 중요한 행동 | 미정 | 미정 |
| 도전 요소 | 행동을 시험하는 위험 또는 문제 | 미정 | 미정 |
| 피드백 | 행동 결과를 읽게 하는 표현 | 미정 | 미정 |
| 보상 / 진행 | 반복을 지속하게 하는 보상 | 미정 | 미정 |
| 목표 조건 | 짧은 반복의 종료 조건 | 미정 | 미정 |

## 아트 요구사항

핵심 반복 구조의 각 상태는 시각적으로 읽혀야 한다.

- 플레이어 행동의 가독성: 미정
- 도전 요소의 가독성: 미정
- 보상의 가독성: 미정
- 목표의 가독성: 미정

## 개발 요구사항

- 입력/조작 기본 구조: 미정
- 상태 모델: 미정
- 게임플레이 이벤트 흐름: 미정
- 데이터 작성 방식: 미정
- 디버그/검증용 hook: 미정

## UX 요구사항

- 플레이어가 다음 행동을 알 수 있는 단서: 미정
- 즉시 피드백: 미정
- 실패 피드백: 미정
- 목표 달성 피드백: 미정

## QA 시나리오

- 플레이어가 다음 행동을 이해할 수 있다.
- 플레이어가 핵심 행동을 수행할 수 있다.
- 도전 요소가 규칙대로 반응한다.
- 보상 또는 진행 상태가 보인다.
- 목표 조건에 도달할 수 있다.

## 첫 플레이 가능 버전의 최소 범위

첫 플레이 가능 버전은 핵심 반복 구조 전체의 최소 증명이어야 한다.

필수 항목:

- 하나의 플레이어 핵심 행동
- 하나의 도전 요소
- 하나의 피드백 또는 보상
- 하나의 목표
- 하나의 검증 경로
