# 작업 항목

이 디렉터리는 시스템 설계에서 파생된 실제 제작 작업을 보관한다.

중요:

> 작업 항목은 게임 제작의 시작점이 아니라 게임디자인/시스템 분해의 결과다.

## 작업 항목 종류

| 종류 | 목적 |
|---|---|
| 기획 작업 | 규칙, 수치, 콘텐츠 조건 정의 |
| 아트 지시서 | 그래픽, 애니메이션, 이펙트, UI, 사운드 산출물 정의 |
| 개발 작업 | 언리얼 구현 작업 정의 |
| UX 작업 | 입력, 피드백, 화면 흐름 정의 |
| QA 시나리오 | 검증 경로와 완료 기준 정의 |
| 기능 계약 | 여러 작업을 묶는 작은 구현 계약 |

## 생성 조건

작업 항목은 아래 정보가 있어야 생성한다.

- 출처 게임 설계 원칙
- 출처 시스템
- 출처 시스템 구성요소
- 대상 세로 단면 / 버전
- 검증 시나리오

## JSON Mirror

필요한 경우 작업 항목 또는 기능 계약을 JSON 복사본으로 만들 수 있다. 다만 사람이 검토하는 기본 문서는 마크다운이다.
